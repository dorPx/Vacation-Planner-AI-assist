# Product

## Register

product

## Users

Travelers planning a full trip in one sitting: they search a destination with dates, compare real hotels/flights/activities/restaurants pulled live from multiple providers, then assemble and export an itinerary. Context: desktop-first research sessions, often with a map open; mobile for quick lookups. The job to be done is "find the best real options fast and trust the prices."

## Product Purpose

Voyager is a travel meta-search and itinerary builder — a booking.com/TripAdvisor-class results experience backed by live scrapers (TripAdvisor, Booking.com, Google Places, Google Flights) plus an AI planner that drafts day-by-day itineraries. Success = a search returns dense, real, comparable results quickly, and a user leaves with a saved/exported itinerary.

## Brand Personality

Trustworthy, efficient, warm-professional. The interface should feel like a working travel tool, not a startup brochure: information-dense but calm, with clear prices and ratings a user can act on immediately.

## Anti-references

- Generic AI-SaaS landing gloss (hero metrics, gradient text, glass cards).
- Marketing-page emptiness — this is a results tool; screens should be full of real data, not slogans.
- Fake polish over broken data: never display invented prices, padded ratings, or placeholder listings as if real.

## Design Principles

1. **Results-first speed** — the search-to-results path is the product; every second of wait and every layout jump there costs trust.
2. **Honest data** — mixed-scale ratings normalized, missing prices labeled, sources attributed; degrade gracefully when a provider fails.
3. **Density with hierarchy** — booking.com-grade information density, organized so price/rating/CTA are scannable in one pass.
4. **Every result actionable** — view deal, compare, locate on map, add to itinerary; no dead-end cards.
5. **One scroll, no traps** — the page flows top to bottom; no nested scrollbars or overlaying panels fighting the document.

## Accessibility & Inclusion

WCAG 2.1 AA target: 4.5:1 body-text contrast, keyboard-operable controls (tabs, filters, compare), `prefers-reduced-motion` alternatives for all animation, alt text on listing images.
